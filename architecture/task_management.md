# SOP: Task Management (Управление задачами)

## Цель
Обеспечить создание, хранение, обновление и удаление задач пользователя в Supabase. Гарантировать целостность данных и соблюдение бизнес-правил (макс. 7 задач/день, конкретные формулировки, привязка к пользователю).

---

## Входные данные
- `user_id` — UUID пользователя (из Telegram chat_id → mapping в users)
- `title` — текст задачи (обязательно, конкретное действие)
- `description` — детали (опционально)
- `date` — дата задачи (по умолчанию: сегодня)
- `priority` — high / medium / low (по умолчанию: medium)
- `estimated_minutes` — оценка времени (по умолчанию: 25)

## Выходные данные
- Подтверждение создания/обновления/удаления
- Список задач с фильтрацией
- Статистика по задачам

---

## Бизнес-правила

1. **Лимит задач:** максимум 7 задач со статусом `pending` или `in_progress` на одну дату
2. **Конкретность:** задача не может быть пустой или < 3 символов
3. **Статусы:** `pending` → `in_progress` → `done` | `skipped`
   - Обратный переход: `done` → `pending` допускается (отмена выполнения)
   - `skipped` → `pending` допускается
4. **Привязка:** каждая задача принадлежит конкретному `user_id`
5. **Время:** `estimated_minutes` от 5 до 240 (жёсткие границы)
6. **Дата:** нельзя создавать задачи в прошлом (предупреждение, но не блокировка)

---

## Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| Превышен лимит 7 задач/день | Отказ + сообщение «Лимит на день исчерпан. Перенесите задачу на другой день.» |
| Пустой title | Отказ + сообщение «Укажите конкретное действие» |
| Дубликат title на тот же день | Предупреждение, но создание разрешено |
| Задача не найдена по ID | Ошибка «Задача не найдена» |
| Supabase недоступен | Логировать ошибку, вернуть сообщение «Сервис временно недоступен» |
