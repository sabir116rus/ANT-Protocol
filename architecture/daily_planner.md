# SOP: Daily Planner (Ежедневный планировщик)

## Цель
Формировать и доставлять структурированный план задач на день через Telegram. Обеспечить утреннюю рассылку плана и контроль лимитов.

---

## Входные данные
- `user_id` — UUID пользователя
- `date` — дата плана (по умолчанию: сегодня)
- Задачи из Supabase со статусом `pending` на указанную дату

## Выходные данные
- Форматированное Telegram-сообщение с планом дня
- Запись в `daily_reports` (тип: `morning_plan`)

---

## Бизнес-правила

1. **Время рассылки:** утренний план отправляется ежедневно (настраивается через n8n Cron)
2. **Сортировка:** задачи отсортированы по приоритету: high → medium → low
3. **Лимит:** показывает максимум 7 задач
4. **Пустой план:** если задач нет → «На сегодня задач нет. Добавьте задачу через /add»
5. **Перенесённые задачи:** задачи с прошлых дней со статусом `pending` автоматически показываются с пометкой ⚠️
6. **Формат времени:** показывать общую оценку времени на день

---

## Формат сообщения (Telegram Markdown)

```
📅 *План на 2026-04-06*

🔴 1. Написать функцию авторизации (30 мин)
🔴 2. Исправить баг в checkout (25 мин)
🟡 3. Изучить документацию Stripe API (25 мин)
🟢 4. Обновить README проекта (15 мин)
⚠️ 5. [перенесено] Ответить клиенту (10 мин)

━━━━━━━━━━━━━━━
📊 Задач: 5/7 | ⏱ ~1ч 45мин
```

Приоритеты: 🔴 high | 🟡 medium | 🟢 low

---

## Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| Нет задач на сегодня | Отправить мотивационное сообщение + подсказку /add |
| Задач > 7 | Показать первые 7, предупредить о перегрузке |
| Перенесённые задачи из прошлых дней | Показать с пометкой ⚠️, предложить /done или /skip |
| Telegram API недоступен | Логировать, повторить через 5 мин (макс 3 попытки) |
| Supabase недоступен | Логировать ошибку, не отправлять пустой план |

## Успешный результат
- Утренний план доставлен в Telegram
- Задачи отсортированы по приоритету
- Перенесённые задачи помечены
- Запись создана в `daily_reports`
- Действие залогировано в `activity_logs`
